资源获取 ​

资源获取是 PanSave 的主要工作区。它不是单个功能，而是一组互相配合的入口：先准备转存策略，再决定哪些频道、榜单或条目需要被搜索和跟踪。

转存策略 ​

入口：资源获取 / 转存策略

转存策略决定“资源存到哪里”。

你需要为网盘配置可用目录，并选择一个整理源目录。频道扫描、监控策略、豆瓣求片和 TMDB 看板即时转存都会使用整理源目录。

如果某个网盘没有整理源目录，它仍然能显示在配置里，但不会成为可用的自动转存目标。

监控策略 ​

入口：资源获取 / 监控策略

监控策略决定自动流程怎么跑：

  • 是否启用监控总开关
  • 多久刷新一次
  • 转存之间隔多久
  • 使用哪些搜索来源
  • 每个来源转存到哪个网盘
  • 新条目是否一定先搜索

如果你使用榜单订阅、TMDB 看板自定义监控、豆瓣求片或缺集补救，通常都需要先配置监控策略。

频道扫描 ​

入口：资源获取 / 频道扫描

频道扫描适合处理频道历史消息。

推荐流程：

  1. 选择要扫描的频道。
  2. 给频道关联一个网盘账号。
  3. 设置关键词、屏蔽词和单轮新增限制。
  4. 执行扫描。
  5. 查看扫描结果。
  6. 确认无误后执行转存。

频道扫描分成“扫描”和“转存”两个手动步骤。这样可以避免关键词设置不合适时，系统自动把不需要的资源也存进网盘。

频道监控 ​

入口：资源获取 / 频道监控

频道监控用于持续观察某些频道的新消息。它和频道扫描不同：

  • 频道扫描偏向“扫描历史消息”
  • 频道监控偏向“持续看新消息”

频道监控使用监控策略里的搜索来源和转存目标。没有在监控策略里启用的来源，不能直接在频道监控里使用。

TMDB 看板 ​

入口：资源获取 / TMDB看板

TMDB 看板适合浏览电影和剧集：

  • 按热度、地区、类型、年份筛选
  • 搜索具体片名
  • 查看是否已经加入监控
  • 手动加入或取消监控
  • 对某个条目即时搜索资源

加入监控时，系统使用 TMDB 的准确 ID，不靠片名模糊猜测，能减少同名资源加错的概率。

榜单订阅 ​

入口：资源获取 / 榜单订阅

榜单订阅适合长期追踪热门内容。当前可按来源查看和订阅榜单，例如猫眼、豆瓣、TMDB。

启用榜单后，榜单条目会进入监控清单。后续系统会根据监控策略配置的搜索来源继续寻找资源。

如果只想临时关注某个资源，也可以在榜单卡片或 TMDB 看板里直接加入自定义监控。

监控清单 ​

入口：资源获取 / 监控清单

监控清单是所有自动跟踪条目的汇总视图。你可以在这里看到：

  • 条目来自榜单、自定义、豆瓣求片还是缺集补救
  • 当前是否已入库
  • 是否已入库但缺集
  • 最近搜索和转存历史
  • 手动刷新 TMDB / Emby 状态

监控清单不是频道配置页，也不是榜单配置页。它更像“当前正在跟踪什么”的总表。

常见状态 ​

状态含义
未入库Emby 里还没有确认找到这个资源。
完整入库Emby 已经确认存在，并且剧集不缺集。
已入库但缺集Emby 里有这个剧集，但按 TMDB 标准季集看仍缺少部分集数。
转存失败本次转存没有成功，需要看日志或处理历史。
容量不足网盘空间不足，清理空间后再手动重试。

先从哪一个功能开始 ​

如果你刚部署完成，可以先选择自己最常用的一条链路验证配置是否可用：只需要整理频道历史时看频道扫描，需要追踪热门内容时看榜单订阅，需要处理片单需求时看豆瓣求片，需要补齐剧集时看 Emby 缺集扫描。
