Emby 缺集扫描 ​

Emby 缺集扫描用于检查媒体库里“已经入库但还缺集”的剧集。它适合做周期性补漏，而不是替代订阅工作流的实时状态。

入口：资源整理 / Emby 缺集扫描

使用前准备 ​

需要先配置：

  • Emby 服务地址和 API Key
  • TMDB API Key

缺集判断依赖 TMDB 的标准季集结构。剧集如果没有正确的 TMDBID，系统就很难稳定判断到底缺了哪些集。

扫描流程 ​

  1. 选择 Emby 服务器。
  2. 选择要扫描的媒体库。
  3. 启动扫描。
  4. 等待扫描完成或手动停止。
  5. 查看缺集结果和失败项。
  6. 将需要补找的条目加入监控。

扫描可能比较耗时，尤其是媒体库很大时。页面会显示当前进度和阶段。

结果怎么理解 ​

结果含义
已入库但缺集Emby 里已有该剧集，但按 TMDB 标准集数看仍缺少部分集数。
扫描失败当前条目没能完成检查，需要查看失败原因或稍后重试。
无法加入监控通常是缺少 TMDBID 或类型信息，系统无法确认要监控哪个标准条目。

页面展示的是扫描快照，不是实时 Emby 状态。Emby 后续入库变化，需要重新扫描或通过监控清单刷新确认。

加入监控 ​

缺集条目可以加入监控。加入后，它会进入监控清单，由监控策略继续寻找缺失资源。

支持两种方式：

  • 单条加入监控
  • 批量把当前筛选结果中可加入的条目加入监控

批量加入只会新增尚未监控的条目，不会把已经监控的条目删除。

什么时候重新扫描 ​

建议在这些场景重新扫描：

  • Emby 新增了很多剧集
  • 最近补齐了资源，想确认缺集是否减少
  • TMDB 元数据修正后，需要重新计算
  • 上一次扫描失败较多

如果只是想查看当前监控条目的状态，优先去 资源获取 / 监控清单 手动刷新，不一定要跑全量缺集扫描。

注意事项 ​

  • 同一时间只能运行一个缺集扫描任务。
  • 扫描过程中可以停止，但已经写入的部分结果会保留为当前快照。
  • 缺集扫描不会直接覆盖监控清单里的入库状态。
  • 删除 Emby 配置时，如果只剩缺集扫描快照引用，系统会允许你确认清理扫描历史后再删除；如果还有豆瓣求片、监控历史等真实业务引用，仍会硬阻断。
