常见问题 ​

默认访问地址是什么 ​

默认端口是 16868。

本机访问：

text
http://localhost:16868

服务器或 NAS 访问时，把 localhost 换成设备 IP 或域名。

为什么 Compose 示例使用 host 网络 ​

host 网络下，应用监听的端口就是宿主机端口，不需要写 ports 映射。PanSave 后续可能会有更多本机端口能力，例如 302 反代相关功能，host 网络更省维护。

如果你的 Docker 环境不支持 host 网络，可以按 Docker 部署 里的端口映射示例改回 bridge 模式。

需要备份哪些文件 ​

至少备份整个 data 目录。

里面保存系统配置、登录状态、Telegram 会话和日志。迁移机器时，建议把整个 data 目录一起带走，不要只挑单个文件。

修改密码后为什么要重新登录 ​

登录状态会绑定当前管理员账号和密码版本。修改密码后，旧登录状态会失效，这是为了避免旧会话继续可用。

频道已经添加了，为什么频道扫描里没有任务 ​

常见原因：

  • 当前 Telegram 模式不一致，例如添加的是 API 频道，但现在切到公开频道模式。
  • 频道没有在频道扫描里启用。
  • 频道没有关联网盘账号。
  • 关联的网盘没有整理源目录。
  • 网盘账号已禁用或失效。

频道资源池只是可用列表，具体是否参与扫描要在频道扫描里单独配置。

监控策略里启用了频道，为什么豆瓣求片仍然不搜索 ​

先检查：

  • 当前 Emby 是否选对。
  • 监控策略总开关是否开启。
  • 搜索来源是否在当前 Telegram 模式下可见。
  • 来源是否关联网盘，网盘是否有整理源目录。
  • 豆瓣求片总开关是否开启。

豆瓣求片继承监控策略来源，不读取频道扫描的启用状态。

为什么扫描到了资源但没有自动转存 ​

频道扫描默认拆成两步：先扫描，再手动转存。这是为了让你先确认关键词和命中结果，避免错误资源被自动存入网盘。

监控策略、榜单订阅、豆瓣求片这类自动流程才会按规则自动转存。

Emby 缺集扫描和监控清单是什么关系 ​

缺集扫描是一次快照，用来找出当前媒体库里可能缺少的剧集。监控清单是长期跟踪列表。

你可以把缺集扫描结果加入监控清单，但缺集扫描本身不会直接覆盖监控清单的状态。

代理会影响哪些功能 ​

代理只影响 Telegram 和 TMDB。

豆瓣、Emby、网盘接口等请求不使用这里的代理。如果某个 Emby 地址在内网，确保部署 PanSave 的机器能直接访问它。

授权码保存失败怎么办 ​

先确认授权码复制完整、当前机器可以访问互联网。如果页面仍提示失败，保留错误提示或截图，到 Telegram 群组 联系管理员排查。

授权码从哪里获得 ​

可以通过 捐助支持 获取授权码感谢回馈。捐助用于鼓励开发和支持项目维护，不是购买资源或资源服务。
