授权码使用 ​

授权码用于激活 PanSave 系统。它是对捐助支持者的感谢回馈，首次登录后按页面提示填写即可。

什么时候需要填写 ​

首次登录系统后，如果当前系统还没有授权，会自动进入“授权设置”页面。

你也可以在授权失效或需要重新填写时，按页面提示重新进入授权设置。

使用步骤 ​

  1. 完成 Docker 部署并打开系统。
  2. 使用默认账号登录，首次登录后建议先修改用户名或密码。
  3. 按页面提示进入“授权设置”。
  4. 粘贴获得的授权码。
  5. 点击保存，等待页面提示授权结果。
  6. 授权通过后，侧边栏版本号旁会显示“已激活”。

授权通过后，再继续配置 Telegram、网盘、TMDB、Emby、通知等功能。

授权码从哪里获得 ​

你可以通过 捐助支持 获取授权码感谢回馈：

捐助方式感谢回馈
捐助 50 元永久授权码一枚
捐助 199 元永久授权码一枚 + 3 天一对一远程部署指导

这里的捐助用于鼓励开发和支持项目维护，授权码属于感谢回馈，不是软件售卖或资源服务。

更换设备或重装时怎么办 ​

如果你要迁移到新机器，或重装 Docker 环境，建议先备份部署目录里的 data 文件夹。它保存了系统配置、登录状态和运行记录。

迁移时把 data 文件夹一起带到新环境，再启动 PanSave。这样可以尽量保留原有配置和授权状态。

常见问题 ​

授权码填在哪里 ​

在 PanSave 的“授权设置”页面填写。正常使用时只需要在页面里粘贴授权码并保存。

授权通过后还要配置什么 ​

授权只代表系统可以正常使用，不会自动配置 Telegram、网盘、TMDB 或 Emby。后续仍需要按自己的使用场景完成基础配置。

换机器后怎么办 ​

优先把原来的 data 文件夹整体迁移到新机器，再启动系统。如果没有迁移原数据，需要重新填写授权码并等待授权结果。

保存授权码失败怎么办 ​

先确认当前机器能正常访问互联网，并检查授权码是否复制完整。如果仍然失败，可以保留页面提示或截图，在 Telegram 群组 联系管理员排查。
