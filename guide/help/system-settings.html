系统配置 ​

系统配置是 PanSave 的基础。配置完成后，频道扫描、监控策略、豆瓣求片和缺集扫描才知道“从哪里找、存到哪里、用哪个媒体库确认结果”。

网盘配置 ​

入口：系统配置 / 网盘配置

这里添加 123 云盘或 115 网盘账号。添加后还需要进入 资源获取 / 转存策略 配置目录，否则系统只知道账号存在，不知道资源要存到哪里。

建议做法：

  1. 先添加账号。
  2. 确认账号状态正常。
  3. 到转存策略里添加目录。
  4. 给每个要自动接收资源的网盘选择一个“整理源目录”。

同一个系统最多维护 10 个网盘账号。超过上限后，运行中的转存流程会被阻断，设置页仍允许你删除或调整配置。

代理配置 ​

入口：系统配置 / 代理配置

代理只用于 Telegram 和 TMDB。豆瓣、Emby、网盘接口等请求不会复用这里的代理。

如果你的服务器本身可以直连 Telegram 和 TMDB，可以不配置代理。开启代理时，需要同时填写地址和端口。

Telegram 配置 ​

入口：系统配置 / Telegram 配置

这个页面包含三件事：

  • Telegram 账号登录
  • 内置机器人配置
  • 频道资源池维护

Telegram 账号 ​

使用 Telegram API 模式时，需要填写 API ID、API Hash 和手机号，再按页面提示完成验证码登录。

修改 API ID、API Hash 或手机号会重置登录态，这是正常行为。只改代理或重复保存相同配置，不会强制重新登录。

内置机器人 ​

内置机器人用于：

  • 给管理员发送系统提醒
  • 执行机器人转存
  • 发送部分任务反馈

机器人需要填写 Bot Token，并绑定系统使用的 Telegram 用户 ID。系统通知会优先通过机器人发给这个用户。

频道资源池 ​

频道资源池只是“可用频道列表”。加入频道后，并不代表它一定会参与频道扫描或豆瓣求片。

后续还需要在这些入口单独启用：

  • 频道扫描
  • 监控策略
  • 频道监控
  • 榜单订阅

这样做是为了让同一个频道可以服务不同场景，也避免某个频道被误加入所有自动任务。

邮箱配置 ​

入口：系统配置 / 邮箱配置

邮箱用于发送：

  • 管理员系统通知
  • 豆瓣求片用户通知
  • 任务阻断提醒

SMTP 地址、端口、发件邮箱需要填写完整。保存前建议先用“测试发送”确认能正常发信。

免打扰只影响邮件，不影响 Telegram 机器人通知。

媒体服务配置 ​

入口：系统配置 / 媒体服务配置

这里配置 TMDB 和 Emby。

配置用途
TMDB确认电影、剧集、年份、海报和标准季集结构。
Emby确认资源是否已入库，是否已经缺集。

新增或编辑 TMDB、Emby 时，系统会先测试连通性和凭据是否有效，测试失败不会保存。

Emby 最多允许配置 5 个。多 Emby 场景下，监控策略、榜单订阅、豆瓣求片和缺集扫描都会按当前选中的 Emby 服务器隔离。

账号和授权 ​

首次登录后建议在右上角“账号设置”里修改默认用户名或密码。

授权码在“授权设置”页面填写。授权有效时，侧边栏版本号旁会显示“已激活”。详细步骤见 授权码使用。

删除配置时的保护 ​

网盘账号、内置机器人、Emby、TMDB 等基础配置如果已经被业务使用，删除时会被阻断。

常见例子：

  • 网盘账号被频道扫描或监控策略引用
  • Emby 被豆瓣求片、榜单订阅或缺集扫描引用
  • 内置机器人被转存策略引用

遇到阻断时，按弹窗提示先去对应功能解除引用或更换配置，再回来删除。系统不会为了删除基础配置而偷偷清理业务数据。
